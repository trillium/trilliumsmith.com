I started using AI coding agents beginning this year, 2024. The first iteration was interacting with the chat version of ChatGPT, which wasn't that great of an experience. Pretty quickly after that I switched to a Pro subscription of CoPilot and that really kicked things off. The integration of an AI agent into the code terminal was the glue I had needed to get better results out, and I've been using human-in-the-loop agentic programming since. It's only gotten better since I started.

## The Landscape

Available now is Claude Code, automated code spaces, long running agentic workflows on cloud infrastructure, MCP servers.

I am now consistently using Claude Code in the terminal, and commonly multiple instances at the same time. Using tools like happy coder also unblocks me for long running tasks because happy coder allows me to give Claude the authorization it needs to continue on tasks.

To my knowledge this option of multiple agents and confirmation from afar wasn't available until early this year. It's made a world of difference in my ability to plan and deliver features faster.

## The Agent Ecosystem

### Code Agents

#### GitHub Copilot Pro

This tool is not as common in my workflow these days because I've switched to using Claude, but it was essential in getting me to figure out how to work on things. Copilot was my first introduction to using Anthropic Sonnet models, which was the toe tip in the pool that I needed.

I still have a Pro subscription for the remainder of the year. I will probably not renew it, however there are now some good agentic development features such as PR reviews.

#### Claude Code

Initial use was in Copilot, integrated into VSCode. Now I use the terminal version instead of the extension, commonly in the integrated terminal, or moving a terminal window into the code pane so that I can have a convenient split view. I run Claude Code via Happy Coder, which will expose those same terminal instances to a web flow or a mobile application.

I use Claude Code to help me organize my understanding of a code base, asking questions like "create me a file of the authentication flow, including file names, function names, and the types the functions use" to give me a better understanding of where different parts of the code base reside. I find this a lot quicker than having to do all these searches myself, and this ability has provided me enough context to build and develop features in new codebases quickly.

#### Opencode.ai

Opencode.ai is a terminal UI tool to guide a GPT model in running workflows and actions. It's open source, and plugs nicely With services that provide subscriptions to LLM services like copilot,. This tool ticks over the responsibility of feeding the language model the tasks it running, as well as when to compact, et. It is nice as an option for getting more out of Chatgpt however at this time I view Claude Code as a superior tool.

### MCP Tools Integration

#### Off-the-Shelf Tools

**Playwright MCP**

I use playwright2 help debug and troubleshoot frontend issues. Playwright has access to chromium witch can then read different property states on the DOM to confirm that cs is styling properly, as well as automate key presses and workflows so the MCP server can be directed2 do custom actions, as well as pipe screenshot images back to the coding agent for a process of development wear the agent can confirm visual elements are on the screen before running another test. This tool has been extremely helpful and me getting to the point where I can essentially use dictation software to get far in my debugging process. If it fails I will get closer into the code to try to discover the problem, but oftentimes spoken word describing the issue to the coding agent is enough to get it to resolve.

**PostHog MCP**

MCP tools like the post hog tool provide rappers around the apis so that we can ask an agent to query states and then the agent turns those queries into something that the pi can understand. I use post hog mcp to interact with their pi and kit me information on my analytics for the services that I run. Post hog also has additional features like feature flags but I haven't integrated yet, but I plan on using mcp services to trigger those as well once they're integrated into my applications

#### Custom-Built MCP Tools

**Google Calendar Access**

I use my google calendar access MCP tool to query my upcoming events, and then create new ones based on strings I provide to the coding agent. For example, recently I wanted to have a notification for a book that I'm interested in ( the second novel in the seriesThe Captive's War), and I used the MCP server to achieve that.

**Gmail Access**

I use gmail MCP to search my emails for specific search strings that have been sent to me from automated services, and then use those emails to verify data in other applications. For example I use the gmail map to get appointment emails from a calendar application and then send them to a session tracking and payment database
